## 機能: ログイン
### 概要
（1〜3行）

### 画面要素（Compose 画面。主要要素は testTag 候補も併記）
- ID/パスワード入力欄（login_email_input / login_password_input）
- ログインボタン（login_button）
- エラーメッセージ表示領域（login_error_text）

### ユーザー操作と期待挙動
- 正しい ID/パスワードを入力してログインボタン押下 → ダッシュボードへ遷移
- 不正な ID/パスワード → エラーメッセージ表示

### エラーケース
- ID 空欄 → 「ID を入力してください」
- パスワード空欄 → 「パスワードを入力してください」
- オフライン時 → 「ネットワークに接続できません」（機内モードで確認）
- 画面回転・バックグラウンド復帰で入力内容が保持されること

### 受け入れ条件
- Given: 未ログイン状態
  When: 正しい ID/パスワードでログイン
  Then: ダッシュボードに遷移する

## 5. データモデル
- User: id, email, password_hash, created_at
- Product: id, name, price, stock, created_at
- Order: id, user_id, total_amount, status, created_at
- OrderItem: id, order_id, product_id, quantity, unit_price

## 6. 非機能要件
- 性能: 起動からホーム画面表示 < 3秒（エミュレータ基準）
- オフライン: ローカルデータ（Room）は機内モードでも閲覧可能
- 権限: 必要最小限（使用する場合は目的を明記。例: カメラ＝書類撮影）
- セキュリティ: 秘密情報は DataStore/EncryptedStorage に保存（平文ファイル禁止）

## 7. 制約事項
- UI: Kotlin + Jetpack Compose（Material 3）
- DB: Room（端末内 SQLite）/ 設定: DataStore
- API 連携が必要な場合: Retrofit（接続先 URL は設定画面等で変更可能にする）
- 対応 OS: Android（minSdk は特段の要件が無ければ雛形既定に従う）

## 8. 受け入れ条件（システム全体）
- 全機能の受け入れ条件が PASS
- 全主要動線がエミュレータ/実機（Maestro）で通り、logcat にクラッシュが無い
- 各ペルソナの目的達成型シナリオが完走する
