# 美少女連鎖パズル（仮）要件定義書

## 1. 目的・背景

本システムは、Android スマートフォン向けに「ぷよぷよ」型の落下連鎖パズルゲームを提供することを目的とする。市販落ちゲーの一般的機能（セーブ／ロード、難易度選択、音声 ON/OFF など）を備えつつ、以下の3点を独自価値として強く打ち出す。

1. **連鎖時の極めて派手なビジュアル・モーション・効果音・BGM 演出** により、既存ゲームを超える爽快感をユーザーに与える。
2. **Docker 上の画像生成 AI（ComfyUI）で生成したオリジナル美少女キャラクター**を多数収録し、キャラクターごとに複数の立ち絵（ポーズ・表情）バリエーションを持たせる。
3. **Docker 上の VoiceBOX で生成した可愛らしいキャラクターボイス**で、操作・連鎖・勝敗の各場面を彩る。

最終的なゴールは、プレイ動画やキャプチャを SNS にアップした際に「グラフィカル・動きが派手・キャラクターが美少女」の3要素でバズる・話題になることである。

## 2. 対象ユーザー

ペルソナ候補（システム設計フェーズ S3 で複数ペルソナに展開する）。

- **P1: パズルゲーム愛好家（20〜30代、ITリテラシー中〜高）**
  - 「ぷよぷよ」「テトリス」「ツムツム」などの連鎖系/落ちゲーを日常的にプレイ。
  - 高難度モードで連鎖を組むこと自体を楽しむ。
- **P2: SNS で映えるゲームを探しているライト〜ミドル層（10代後半〜20代、ITリテラシー中）**
  - TikTok / X / YouTube Shorts に短尺動画を上げて反応を見るのが好き。
  - 大連鎖発生時の派手な演出と美少女キャラクターをシェアしたい。
- **P3: 美少女キャラクター・ボイス目当てのユーザー（年齢層幅広、ITリテラシー中）**
  - ストーリーモードでお気に入りキャラと触れ合いたい。
  - キャラのボイスや立ち絵の差分が増えるほど嬉しい。
- **P4: 隙間時間のカジュアル層（年齢層幅広、ITリテラシー低〜中）**
  - 数分の通勤・休憩中にエンドレスモード／スコアアタックで気軽に遊ぶ。
  - 操作はシンプルで、難易度 Easy から始めたい。

## 3. システム全体像

- **クライアント（Android アプリ本体）**
  - Kotlin + Jetpack Compose（Material 3）による単一バイナリ。縦持ち（Portrait）固定。
  - ゲームロジック、描画（連鎖エフェクト含む）、サウンド再生、セーブ／ロード、ローカルランキングを内包。
  - 画像・音声アセットは APK にバンドル済み（オフライン動作）。
- **アセット生成パイプライン（開発時のみ・Docker 上で稼働）**
  - **ComfyUI（画像生成 AI）**: インターネットから取得したベース画像を入力に、オリジナルの美少女キャラクター立ち絵を生成。1キャラあたり 10 パターン（通常／喜／怒／哀／連鎖中／大連鎖／敗北）を生成する。
  - **VoiceBOX（音声合成）**: キャラクターごとに台本に従って可愛らしい音声を事前生成する。
  - 生成された画像・音声は最適化（解像度・音質圧縮）の上、`app/src/main/res/` または `assets/` に取り込み、リリース APK にバンドルする。
- **端末内永続化**
  - **Room（SQLite）**: ローカルランキング（TOP10）、ストーリーモード進行状況、セーブデータ。
  - **DataStore**: 設定（難易度・音量・BGM/SE ON/OFF・選択キャラなど）。
- **外部連携**
  - シェア機能用に Android 標準 `ACTION_SEND` を介して、OS のシェアシートに画像／動画を渡す。専用バックエンドは持たない。

## 機能: タイトル / メインメニュー

### 概要
アプリ起動直後に表示されるトップ画面。各モードや設定への入口を提供する。

### 画面要素（Compose 画面。主要要素は testTag 候補も併記）
- タイトルロゴ表示（title_logo）
- 「はじめる」ボタン（title_start_button）
- 「キャラクター選択」ボタン（title_character_select_button）
- 「ランキング」ボタン（title_ranking_button）
- 「チュートリアル」ボタン（title_tutorial_button）
- 「設定」ボタン（title_settings_button）
- 背景アニメーション・選択中キャラの立ち絵プレビュー（title_character_preview）

### ユーザー操作と期待挙動
- 「はじめる」押下 → ゲームモード選択画面へ遷移
- 「キャラクター選択」押下 → キャラクター選択画面へ遷移
- 「ランキング」押下 → ローカルランキング画面へ遷移
- 「チュートリアル」押下 → 任意チュートリアル画面へ遷移
- 「設定」押下 → 設定画面へ遷移

### エラーケース
- 初回起動でセーブデータ未生成 → 自動的に空セーブを生成して進行可能にする
- 画面回転・バックグラウンド復帰で BGM の二重再生が起きないこと

### 受け入れ条件
- Given: アプリ初回起動
  When: タイトル画面が表示される
  Then: すべての主要ボタンが押下可能で、各画面への遷移が成功する

## 機能: 任意チュートリアル

### 概要
メインメニューから任意のタイミングで再生可能なチュートリアル。基本操作・連鎖の組み方を、サンプルプレイ＋字幕＋キャラクターボイスで実演する。初回起動時に強制はしない。

### 画面要素
- チュートリアル本文表示領域（tutorial_text_area）
- 操作デモ表示領域（tutorial_demo_area）
- 「次へ」ボタン（tutorial_next_button）
- 「とばす」ボタン（tutorial_skip_button）
- ページインジケータ（tutorial_page_indicator）

### ユーザー操作と期待挙動
- 「次へ」押下 → 次のチュートリアルページへ進む
- 「とばす」押下 → タイトル画面へ戻る
- 全ページ閲覧完了 → 完了メッセージ → タイトル画面へ戻る

### エラーケース
- 途中でホームに戻り再開時 → 進行ページが復元されること（DataStore に最終閲覧ページを保存）
- ボイス再生中に画面回転 → 再生が止まらず継続すること

### 受け入れ条件
- Given: タイトル画面
  When: 「チュートリアル」ボタンを押下し最後まで進める
  Then: 操作デモ・字幕・ボイスが順次再生され、完了後にタイトル画面に戻る

## 機能: キャラクター選択

### 概要
プレイに使用する美少女キャラクターを選ぶ画面。初回起動時から全 6 キャラが選択可能。選択中キャラはタイトル・各モードに反映される。

### 画面要素
- キャラクター一覧（横スクロール、character_list）
- 各キャラカード（character_card_{characterId}）
- 選択中キャラの大きな立ち絵プレビュー（character_preview_image）
- ボイスプレビュー再生ボタン（character_voice_preview_button）
- 「決定」ボタン（character_confirm_button）

### ユーザー操作と期待挙動
- キャラカードをタップ → プレビューエリアが切り替わり、サンプルボイスが1回再生される
- 「決定」押下 → 選択キャラを DataStore に保存しタイトル画面へ戻る

### エラーケース
- 立ち絵画像の読み込みに失敗 → プレースホルダ画像を表示し、エラー音は出さない
- ボイス再生中に別キャラを選択 → 直前のボイスを停止して新しいキャラのボイスを再生

### 受け入れ条件
- Given: キャラクター選択画面
  When: いずれかのキャラを選び「決定」を押下
  Then: 選択結果が永続化され、再起動後も同じキャラが選択状態である

## 機能: エンドレスモード（プレイ画面）

### 概要
ゲームオーバーになるまで延々と続く基本モード。スコアと連鎖数を競う。

### 画面要素
- プレイフィールド 6 × 14（playfield_grid）
- 落下中ぷよ（playfield_current_pair）
- ネクスト・ネクストネクスト表示（playfield_next, playfield_next_next）
- スコア表示（playfield_score）
- 連鎖数表示（playfield_chain_count）
- 一時停止ボタン（playfield_pause_button）
- 操作 UI: 左移動／右移動／回転／高速落下／ハードドロップ（playfield_btn_left, playfield_btn_right, playfield_btn_rotate, playfield_btn_softdrop, playfield_btn_harddrop）

### ユーザー操作と期待挙動
- 左／右ボタンで落下中ぷよを 1 マス移動
- 回転ボタンで時計回りに 90 度回転
- 高速落下押下で落下速度を上げる、離すと通常速度に戻る
- ハードドロップで即着地
- 3 つ以上の同色ぷよが縦横で隣接 → 消去 → 上のぷよが落下 → 連鎖判定（再帰）

### エラーケース
- 一時停止中にバックグラウンド復帰 → 一時停止状態を維持
- プロセス再起動 → 直近のオートセーブから「中断中」として復帰可能（リジューム選択可）
- 端末スリープ → ゲーム進行を停止し、復帰時に一時停止画面を表示

### 受け入れ条件
- Given: エンドレスモード開始直後
  When: 同色ぷよ 3 つを縦横で隣接させる
  Then: 当該ぷよが消去され、連鎖カウントが増加しスコアが加算される

## 機能: スコアアタックモード（プレイ画面）

### 概要
制限時間内のハイスコアを競うモード。制限時間（例: 3 分）終了時にリザルト画面へ遷移しスコアを記録する。

### 画面要素
- エンドレスモードと同じプレイ画面 UI
- 残り時間表示（scoreattack_timer_text）

### ユーザー操作と期待挙動
- 開始から制限時間カウントダウン
- 操作・連鎖ロジックはエンドレスと共通
- 残り時間 0 → リザルト画面へ自動遷移し、TOP10 入りすればローカルランキングに記録

### エラーケース
- 一時停止中はタイマー停止
- バックグラウンド復帰時もタイマーは停止扱い
- プロセス再起動 → セッションは破棄され、タイトル画面へ戻る（中断スコアは保存しない）

### 受け入れ条件
- Given: スコアアタックモードを開始
  When: 制限時間が経過する
  Then: 自動でリザルト画面に遷移し、スコアと連鎖数がランキング判定に渡される

## 機能: ストーリーモード（プレイ画面 + 会話パート）

### 概要
キャラクターごとの短編シナリオを楽しむモード。会話パート → バトル（エンドレス系プレイ） → 会話パートを繰り返す構成。

### 画面要素
- 会話パート: 立ち絵表示（story_character_image）、台詞テキスト（story_dialog_text）、話者名（story_speaker_name）、「次へ」ボタン（story_next_button）、「ログ」ボタン（story_log_button）
- バトルパート: エンドレスモードと同じプレイ画面 UI + 章タイトル表示（story_chapter_title）

### ユーザー操作と期待挙動
- 「次へ」押下 → 次の台詞へ進む（ボイス自動再生）
- バトル開始 → 章ごとに設定された勝利条件（例: スコア n 点、n 連鎖達成）でクリア判定
- 章クリア → 進行状況を Room に保存し、次の章へ進める

### エラーケース
- ボイスファイル欠落 → 台詞テキストのみ表示し、無音で進行
- バトル中ゲームオーバー → 章選択画面に戻る（進行状況は前章まで維持）

### 受け入れ条件
- Given: ストーリーモード章 1 開始
  When: 会話を進めバトル勝利条件を満たす
  Then: 章クリア演出が表示され、進行状況が永続化される

## 機能: CPU 対戦モード（プレイ画面）

### 概要
ユーザー vs CPU の 1 対 1 連鎖バトル。連鎖を組むと相手側におじゃまぷよを送り込む。

### 画面要素
- 2 つのプレイフィールドを上下に並べる（cpu_player_field, cpu_enemy_field）
- 各フィールド横にスコア・連鎖数・送られたおじゃまぷよ予告表示（cpu_player_ojama, cpu_enemy_ojama）
- CPU キャラ立ち絵表示（cpu_enemy_character_image）
- 操作 UI: エンドレスと同じ

### ユーザー操作と期待挙動
- 自分が n 連鎖を達成 → CPU 側のフィールド上部におじゃまぷよ予告を蓄積
- 次に自分のぷよが着地したタイミングで予告分のおじゃまぷよが CPU 側に降る
- どちらかのフィールドがゲームオーバー条件を満たした時点で勝敗確定 → リザルト画面へ
- 難易度設定により CPU の連鎖力・操作速度が変化

### エラーケース
- 同時にゲームオーバー → ユーザー側を敗北として扱う（CPU 優位の引き分け処理は行わない）
- 一時停止 → CPU 思考も停止する

### 受け入れ条件
- Given: CPU 対戦モードを難易度 Normal で開始
  When: 自プレイヤーが 4 連鎖を達成
  Then: CPU 側におじゃまぷよが送り込まれる演出が発生し、CPU フィールドが実際に埋まる

## 機能: 連鎖演出システム（共通エンジン）

### 概要
連鎖が発生した際に、画面全体で爽快感を最大化する演出を発火する共通エンジン。各プレイ画面（エンドレス／スコアアタック／ストーリー／CPU 対戦）から呼び出される。

### 演出要素
- 消去ぷよ拡大 → 弾けるパーティクルエフェクト（chain_particle_layer）
- 連鎖数表示が大きく中央にズームイン → スケーリング＋カラーフラッシュ（chain_count_overlay）
- 連鎖数に応じた画面シェイク（chain_screen_shake）
- 連鎖数に応じた背景フラッシュ／カラーグラデーション変化（chain_background_flash）
- 連鎖数に応じたキャラクター立ち絵の差し替え（通常 → 連鎖中 → 大連鎖）
- 連鎖数に応じたキャラクターボイス（「いっき！」「にれんさ！」「だいれんさ！」など）
- 連鎖数に応じた SE（小連鎖 SE／中連鎖 SE／大連鎖 SE）
- 5 連鎖以上で BGM が高揚パートにスイッチ（chain_bgm_intensify）

### ユーザー操作と期待挙動
- 連鎖判定がトリガされる → 上記演出が並列発火し、消去アニメーション完了後に次のぷよ落下を再開
- 演出は短時間でループせず、連鎖ごとに段階的に派手さが増す

### エラーケース
- 端末性能不足でフレームレート低下 → パーティクル数を動的に減らす（30 fps を下回らない範囲で）
- BGM スイッチ中に連鎖終了 → 元の BGM パートにフェードバック

### 受け入れ条件
- Given: 任意のプレイ画面で 5 連鎖以上を発生させる
  When: 連鎖判定が走る
  Then: 拡大パーティクル・画面シェイク・大連鎖ボイス・大連鎖 SE・BGM 高揚スイッチが連動して発火する

## 機能: キャラクターボイス・BGM・SE 再生

### 概要
VoiceBOX で事前生成された各キャラのボイスファイル、BGM、SE をアプリ内で管理・再生する共通基盤。

### 画面要素
- 画面はもたず、すべての画面から共通サービスとして呼び出される

### 仕様
- ボイス: キャラごとに 1 アセットファイル群（assets/voice/{characterId}/{eventId}.ogg）
- BGM: モード／シーンごと（assets/bgm/{sceneId}.ogg）
- SE: 操作・連鎖・UI（assets/se/{eventId}.ogg）
- 音量・ON/OFF は設定画面（DataStore）に従う

### エラーケース
- ファイル欠落 → 該当再生をスキップ、ログには警告のみ
- 通話着信 → AudioFocus を譲渡し、復帰時に BGM のみ再生再開（SE/ボイスは再開しない）

### 受け入れ条件
- Given: 設定でボイス ON、BGM ON、SE ON
  When: プレイ中にぷよを操作・連鎖を発生させる
  Then: 操作 SE・連鎖ボイス・連鎖 SE・BGM が混ざって正しく再生される

## 機能: 大連鎖クリップ自動保存（5 連鎖以上）

### 概要
SNS 共有を狙い、5 連鎖以上の演出を発生させた瞬間に、その演出区間（連鎖開始 〜 演出終了）を自動的に短尺動画（GIF または mp4）として端末に保存する。

### 画面要素
- プレイ画面の右上に「保存しました」トースト（chain_clip_saved_toast）
- 設定画面で機能 ON/OFF（settings_chain_clip_toggle）

### ユーザー操作と期待挙動
- 5 連鎖以上を達成 → 演出区間（最大 10 秒）を自動的にキャプチャしギャラリーに保存
- トーストから「シェア」ボタンで OS 標準シェアシートを開ける

### エラーケース
- ストレージ容量不足 → 警告トーストを表示しキャプチャを中止（プレイ自体は継続）
- 端末性能不足 → 保存を断念し警告ログを残す
- 権限拒否（書き込み権限が必要な場合） → 設定で機能を OFF にする旨を案内

### 受け入れ条件
- Given: 設定で大連鎖クリップ自動保存 ON
  When: プレイ中に 5 連鎖を発生させる
  Then: ギャラリーに動画ファイルが保存され、画面に保存トーストが表示される

## 機能: スクリーンショット共有

### 概要
リザルト画面・任意のタイミングで現在画面のスクリーンショットを取得し、OS 標準シェアシートで SNS や保存先に送る最小限のシェア機能。

### 画面要素
- 「シェア」ボタン（share_button）。リザルト画面およびゲーム画面の一時停止メニューに配置。

### ユーザー操作と期待挙動
- 「シェア」押下 → 現在画面をキャプチャ → OS 標準シェアシート起動

### エラーケース
- 一時的に View が描画途中 → キャプチャ完了まで遅延し、失敗時は再試行案内
- 共有先アプリ未インストール → OS シェアシートの標準挙動に委ねる

### 受け入れ条件
- Given: リザルト画面表示中
  When: 「シェア」ボタンを押下
  Then: OS のシェアシートに当該スクリーンショットが添付された状態で表示される

## 機能: ローカルランキング（TOP10）

### 概要
エンドレス／スコアアタック／CPU 対戦の各モード別に、端末内でハイスコア TOP10 を保持する。

### 画面要素
- モード切り替えタブ（ranking_tab_endless, ranking_tab_scoreattack, ranking_tab_cpu）
- ランキング一覧（ranking_list）
- 各行: 順位、スコア、最大連鎖数、使用キャラ、日付（ranking_row_{rank}）
- 「戻る」ボタン（ranking_back_button）

### ユーザー操作と期待挙動
- タイトル → 「ランキング」押下 → ランキング画面表示
- モードタブ切り替えで各モードの TOP10 を表示
- リザルト画面で TOP10 入りした場合は強調表示

### エラーケース
- データ未登録 → 「まだ記録がありません」表示
- スコア同点 → 達成日時が古い順に上位表示

### 受け入れ条件
- Given: エンドレスモードでハイスコア TOP10 入りを達成
  When: タイトル → ランキング画面を開く
  Then: 当該スコアがエンドレスタブの該当順位に表示される

## 機能: セーブ／ロード（オート＋手動スロット）

### 概要
中断復帰用のオートセーブと、複数スロットの手動セーブをサポートする。ストーリーモードでは章進行を別管理する。

### 画面要素
- 設定画面内 / 一時停止メニューに「セーブ」「ロード」項目（save_button, load_button）
- スロット一覧画面（save_slot_list）、各スロット行（save_slot_{index}）
- 「上書き」「読み込み」「削除」ボタン

### ユーザー操作と期待挙動
- プレイ中はオートセーブを定期的（ぷよ着地ごと）に更新
- 一時停止メニューから手動セーブ → スロット選択 → 上書き確認 → 保存
- タイトルから「つづきから」相当を将来追加する場合に備え、オートセーブからの復帰インタフェースを用意

### エラーケース
- スキーマ変更時は Room Migration を実装し、旧データを失わないこと
- セーブデータ破損 → ダイアログで通知し、当該データを破棄して新規プレイ開始へ誘導

### 受け入れ条件
- Given: エンドレスモード中にホームへ戻る
  When: 再度アプリを開き「つづきから」相当の復帰を行う
  Then: 直前のぷよ配置・スコア・連鎖状態が復元される

## 5. データモデル

### Room Entity

- **CharacterEntity**（収録キャラ定義。マスタとして APK に同梱）
  - `id`（PK, String）
  - `displayName`（String）
  - `voiceTone`（String, VoiceBOX 話者種別の識別子）
  - `unlocked`（Boolean, 既定 true）

- **CharacterImageEntity**（キャラ別立ち絵バリエーション）
  - `id`（PK, String）
  - `characterId`（FK → CharacterEntity.id）
  - `variant`（String, 列挙: normal / joy / anger / sad / chain / bigChain / lose）
  - `assetPath`（String, バンドルパス）

- **HighScoreEntity**（ローカルランキング）
  - `id`（PK, Long, autoGenerate）
  - `mode`（String, 列挙: endless / scoreAttack / cpuBattle）
  - `score`（Long）
  - `maxChain`（Int）
  - `characterId`（String）
  - `playedAt`（Long, epoch ms）

- **StoryProgressEntity**（ストーリーモード進行）
  - `id`（PK, String, characterId）
  - `clearedChapter`（Int, 既定 0）
  - `updatedAt`（Long, epoch ms）

- **SaveSlotEntity**（手動セーブスロット）
  - `slotIndex`（PK, Int, 1〜n）
  - `mode`（String）
  - `serializedGameState`（String, JSON）
  - `score`（Long）
  - `savedAt`（Long, epoch ms）

- **AutoSaveEntity**（オートセーブ、単一行）
  - `id`（PK, Int, 固定 0）
  - `mode`（String, nullable）
  - `serializedGameState`（String, nullable）
  - `savedAt`（Long, epoch ms, nullable）

### DataStore キー

- `settings.difficulty`（String, Easy / Normal / Hard / Expert、既定 Normal）
- `settings.bgm.enabled`（Boolean, 既定 true）
- `settings.bgm.volume`（Float 0.0〜1.0, 既定 0.8）
- `settings.se.enabled`（Boolean, 既定 true）
- `settings.se.volume`（Float, 既定 1.0）
- `settings.voice.enabled`（Boolean, 既定 true）
- `settings.voice.volume`（Float, 既定 1.0）
- `settings.chainClip.enabled`（Boolean, 既定 true）
- `selected.characterId`（String, 既定 6 キャラの先頭）
- `tutorial.lastViewedPage`（Int, 既定 0）

### アセット構成（バンドル）

- `assets/image/character/{characterId}/{variant}.webp`（立ち絵 10 パターン × 6 キャラ = 60 枚）
- `assets/voice/{characterId}/{eventId}.ogg`（ボイス）
- `assets/bgm/{sceneId}.ogg`（BGM）
- `assets/se/{eventId}.ogg`（SE）

## 6. 非機能要件

- **性能**
  - 起動からタイトル画面表示 < 3 秒（エミュレータ基準）
  - プレイ中フレームレート 60 fps 目標、最低 30 fps を下回らない（連鎖演出時は動的にパーティクル数調整）
  - 連鎖判定の処理遅延が体感されない（着地から連鎖開始まで 100 ms 以内）
- **オフライン**
  - 画像・音声アセットは APK バンドル済みで、機内モードでも全モードがプレイ可能
  - ローカルランキング・セーブも端末内のみで完結
- **権限**
  - インターネット権限は不要（バックエンドを持たないため）
  - 大連鎖クリップ保存に必要な権限のみ最小限要求（Android のバージョンに応じた MediaStore 利用を前提とし、ストレージ全権限は要求しない）
- **セキュリティ**
  - 設定値・セーブデータは DataStore / Room に保存し、平文ファイルへの保存は禁止
  - 個人を特定する情報（メールアドレス・氏名等）は収集しない
- **アクセシビリティ**
  - 主要 UI 要素には testTag を必ず付与（自動テストおよびスクリーンリーダ補助のため）
  - 文字サイズはシステム設定の倍率に追従

## 7. 制約事項

- **UI**: Kotlin + Jetpack Compose（Material 3）。縦持ち（Portrait）固定。
- **DB**: Room（端末内 SQLite）。設定は DataStore。
- **API 連携**: なし（ローカル完結）。バックエンドサーバは持たない。
- **アセット生成**: Docker 上で稼働する **ComfyUI**（画像生成）と **VoiceBOX**（音声合成）を開発時に用い、生成物は最適化のうえ APK にバンドルする。これらは実行時依存ではない。
- **キャラクター画像のベース**: インターネットから取得したベース画像を ComfyUI ワークフロー（img2img / ControlNet / LoRA）で再構成し、オリジナル化したうえで利用する。各キャラ 10 パターン（通常／喜／怒／哀／連鎖中／大連鎖／敗北）を生成する。
- **対応 OS**: Android（minSdk は特段の要件が無ければ雛形既定に従う）。

## 8. 受け入れ条件（システム全体）

- 全機能の受け入れ条件が PASS する
- 全主要動線（タイトル → キャラ選択 → ゲームモード選択 → 各モード → リザルト → タイトル）がエミュレータ／実機（Maestro）で通り、logcat にクラッシュが無いこと
- 4 つのゲームモード（エンドレス／スコアアタック／ストーリー／CPU 対戦）でそれぞれ 5 連鎖以上を発生させたとき、連鎖演出システムが派手な演出（拡大パーティクル・画面シェイク・大連鎖ボイス・大連鎖 SE・BGM 高揚スイッチ）を発火させ、同時に大連鎖クリップが自動保存される
- 6 キャラそれぞれが選択可能で、立ち絵 10 パターンが連鎖イベントに連動して切り替わる
- 設定画面の難易度／音量／BGM/SE/ボイス ON/OFF トグルが各画面に正しく反映される
- 機内モードでも全モードがプレイ可能で、ローカルランキングに記録できる
- 各ペルソナ（P1 パズル愛好家／P2 SNS 映え狙い／P3 キャラ・ボイス目当て／P4 カジュアル）の目的達成型シナリオが完走する
