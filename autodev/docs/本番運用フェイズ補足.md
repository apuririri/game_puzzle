# AutoDev for Android 運用フェイズ補足

本書は要件定義書 v5.0 Android版（`autodev/docs/要件定義書.md`）の補足である。
開発完了後のアプリを **開発者のスマートフォンへデプロイし、長期運用しながら AutoDev で保守・デプロイ・データ整理を続けられる**よう、以下を追補する。

## 0. 位置づけ・安全境界（NFR-6.5.1 の明確化）

- **AI エージェント（`claude --dangerously-skip-permissions`）は dev / 運用端末側でのみ動かす**。
- 配布されるのは **生成されたアプリ（署名済み APK）だけ**。デプロイは AI が dev 端末から `adb` 越しに行う（ストア公開は対象外）。
- 本ツール本体（`autodev/` `.claude/`）はリリース時に削除可能という分離原則は維持。デプロイ機構も `autodev/` 配下にあり顧客には渡らない。

## 1. 運用対応アプリ（自己完結・実行時設定）

- **DB**: Room（端末内 SQLite）。schema は `app/schemas/` に tracked。**schema 変更時は Migration 必須**（既存ユーザーデータ保全。`check_room_schema.sh` がゲート）。
- **設定**: DataStore。API サーバー URL 等の環境依存値はハードコードせず、設定画面/DataStore から実行時解決。
- **署名**: release は独自 keystore（`autodev/secrets/`、setup.sh が自動生成、git 管理外）。**keystore のバックアップ必須**（紛失すると上書き更新不可）。
- **起動健全性**: Application 起動時に `AppLog: APP_STARTED` を出力（healthcheck / スモークが確認。Web 版 /healthz 相当）。
- **ログ**: 統一タグ（AppLog/ApiLog/DbLog/UiLog）。クラッシュは logcat の FATAL を `logcat_error.sh` で検出。

## 2. デプロイ章（emulator → device）

- 環境は `autodev/config/deploy.yaml` の `environments` に登録（kind=avd | adb-device）。`promotions` で許可フローを定義。
- 起動指示: 「**<source> から <target> へデプロイ**」。skill `autodev-deploy` が D1〜D8 を実行（事前ゲート→アーティファクト→install→スモーク→記録→ロールバック）。
- **build_strategy**（既定 rebuild）:
  - `rebuild`: 現行コードから `build.sh <build_type>` でビルドし `autodev/artifacts/app_<version>.apk` へ世代保管（`artifact_keep` 世代）。
  - `promote`: 直近アーティファクト APK を再ビルドせず採用（検証済み同一バイナリ）。
- **ルール**: `requires_approval=true`（実機 device 既定）は **ユーザーの明示確認**が必須（NFR-6.2.1 の例外）。
  versionCode 後退・署名不一致の install は uninstall（**アプリデータ消失**）が必要なため、必ずユーザー確認。スモーク失敗時は `auto_rollback`。`deploy.sh` は既定 dry-run、`--apply` で実行。
- **スモーク**: `run_smoke_device.sh`（起動マーカー + `maestro/smoke.yaml` + logcat クラッシュスキャン + スクショ）。
- **記録**: `autodev/state/deploys/<target>/<ts>.json`（source/target/git_sha/apk_path/version/smoke/approved）、進捗.md「デプロイ履歴」、CHANGELOG 追記。
- スクリプト: `deploy.sh` `check_deploy_ready.sh` `resolve_artifact.sh` `run_smoke_device.sh` `rollback.sh`。

## 3. データライフサイクル章（洗い替え）

- 長期運用で `autodev/`（state/iterations・attestations・persona_tests・history・evidence・logs・artifacts・inputs/修正対応）が肥大化する。
- 原則: **要約は永続（CHANGELOG.md / docs/変更履歴.md）・詳細はアーカイブ後に prune**。
- 設定: `autodev/config/retention.yaml`（evidence/logs/history/artifacts の保持基準。`evidence/_archived/` は無期限保持）。
- 起動指示: 「**不要データを洗い替えしてください**」。skill `autodev-cleanup` ＋ `cleanup.sh`（既定 dry-run、`--apply` でアーカイブ→prune、`--full-reset` で運用履歴クリーンスレート化）。
- 安全策: `app/ maestro/ docs/設計/ CHANGELOG / docs/変更履歴 / personas` と進行中ループは不可侵。削除前に必ず `autodev/archive/` へ退避。

## 4. 起動プロンプト一覧（拡張後）

| モード | プロンプト例 | skill |
|---|---|---|
| システム全体開発 | 要件定義書.mdのシステムを自動で開発してください | autodev-system-loop |
| 機能単位開発 | 機能要件_<名>.mdの機能を自動で開発してください | autodev-feature-loop |
| 修正対応 | 修正_<名>.mdの修正を自動で対応してください | autodev-fix-loop |
| **デプロイ** | **emulator から device へデプロイしてください** | **autodev-deploy** |
| **洗い替え** | **不要データを洗い替えしてください** | **autodev-cleanup** |

## 5. 中断からの再開（FR-5.8.1 強化）

- **現在地ポインタ `autodev/state/current.json`**: 各ループ/イテレーション手順の冒頭で `autodev/scripts/set_current.sh <mode> <target> <phase> <task> "<step>"` により更新。`{mode,target,loop_phase,current_task_id,last_step,blocked,updated_at}`。
- **再開**: 「作業を再開して」で skill `autodev-resume`。`current.json`＋進捗.md から位置復元 → healthcheck（デバイス無しならエミュレータ自動起動）→ **直前手順を ground-truth 再検証** → 記録位置から継続。`blocked=true` は人手要因として提示・停止。
- SessionStart hook が `check_resume_point.sh` の結果を毎回注入するため、再起動直後に位置を把握できる。

## 6. 無人運用（稼働監視＋自動再開 / opt-in）

- **heartbeat**: `autodev/state/heartbeat.json` を PreToolUse/PostToolUse hook が更新（揮発・gitignore）。AI 停止で stale になる。
- **常駐**: `autodev/scripts/start_session.sh` が `claude --dangerously-skip-permissions` を tmux セッションで起動。
- **watchdog**: `autodev/scripts/watchdog.sh`（cron, `watchdog.yaml` の `interval_min` 毎）が heartbeat stale ＆ `current.json` 進行中＆非 blocked を検知したら、**同一 tmux セッションへ `resume_prompt` を送信**。完了/blocked/質問待ち/レート超過は何もしない。
- **導入（opt-in）**: `autodev/scripts/setup_watchdog.sh`。解除は `teardown_watchdog.sh`。既定は無効。
- **対象環境**: Linux / macOS / WSL（tmux 必須）。
- **停止/一時停止/確認**: 恒久停止は `teardown_watchdog.sh`。一時停止は `watchdog.yaml` の `enabled:false`。確認は `crontab -l | grep autodev-watchdog` / `autodev/logs/watchdog.log` / `tmux ls`。
