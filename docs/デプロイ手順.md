# デプロイ手順（開発者のスマートフォンへ）

本アプリは **開発者のスマートフォンへ adb でインストール**して利用する（ストア公開は対象外）。
release ビルドは独自 keystore で署名する（AutoDev 利用時は `autodev/secrets/` に自動生成済み）。

## 前提

- Android 端末（開発者向けオプション → USB デバッグ有効）
- PC と USB 接続（または Wi-Fi adb: `adb pair`）

## A. AutoDev（AI）でデプロイする場合（推奨）

```text
> emulator から device へデプロイしてください
```

- AI が dry-run → 明示承認の確認 → release ビルド・署名 → adb install → スモーク（起動 + maestro/smoke.yaml + logcat）を行う。
- 失敗時は前世代 APK（autodev/artifacts/）へ自動ロールバック。

## B. 手動でデプロイする場合

```bash
# 1. release APK をビルド（AutoDev 環境なら）
autodev/scripts/build.sh release

# 2. 端末へインストール
autodev/scripts/install.sh app/build/outputs/apk/release/app-release.apk

# （AutoDev を取り外した顧客環境では）
./gradlew assembleRelease
adb install -r app/build/outputs/apk/release/app-release.apk
```

## 注意

- **署名（keystore）を変えると上書きインストールできない**。keystore は必ずバックアップする。
- versionCode を下げた APK は上書きインストールできない（uninstall = アプリデータ消失が必要）。
- Room の schema を変更したリリースは、必ず Migration を実装してから配布する（既存データ保全）。
