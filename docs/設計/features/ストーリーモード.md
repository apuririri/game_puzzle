# ストーリーモード（StoryScreen）

## 機能概要

キャラクターごとの短編シナリオを楽しむモード。会話パート → バトル（エンドレス系プレイ） → 会話パートを繰り返す構成。
1 キャラ 3 章を初期実装（S1 Q2）。

## 画面要素

- 画面ルート（`story_root`）
- 会話パート
  - 立ち絵表示（`story_character_image`）
  - 台詞テキスト（`story_dialog_text`）
  - 話者名（`story_speaker_name`）
  - 「次へ」ボタン（`story_next_button`）
  - 「ログ」ボタン（`story_log_button`）
  - 章タイトル（`story_chapter_title`）
- バトルパート: エンドレスモードと同じプレイ画面 UI（`playfield_*`）

## ユーザー操作

- 「次へ」押下 → 次の台詞へ進む（ボイス自動再生）
- バトル開始 → 章ごとに設定された勝利条件（例: スコア n 点、n 連鎖達成）でクリア判定
- 章クリア → 進行状況を Room（`StoryProgressEntity`）に保存し、次の章へ進める

## エラーケース

- ボイスファイル欠落: 台詞テキストのみ表示し、無音で進行（要件のとおり）。
- バトル中ゲームオーバー: 章選択画面に戻る（進行状況は前章まで維持）

## データモデル

```kotlin
@Serializable
data class StoryChapter(
  val characterId: String, val chapter: Int, val title: String,
  val dialogsBeforeBattle: List<Dialog>,
  val battle: BattleObjective,    // score 目標 or chain 目標
  val dialogsAfterBattle: List<Dialog>
)
data class Dialog(val speakerId: String, val text: String, val voiceEventId: String?)
sealed class BattleObjective { data class Score(val target: Long): BattleObjective(); data class Chains(val target: Int): BattleObjective() }
```

台本は `res/raw/story_<characterId>.json` にバンドル。

## 受け入れ条件

- Given: ストーリーモード章 1 開始
- When: 会話を進めバトル勝利条件を満たす
- Then: 章クリア演出が表示され、`StoryProgressEntity.clearedChapter` が +1 される（Compose UI Test）。
