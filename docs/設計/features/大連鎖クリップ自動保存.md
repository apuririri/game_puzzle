# 大連鎖クリップ自動保存（5 連鎖以上）

## 機能概要

SNS 共有を狙い、5 連鎖以上の演出を発生させた瞬間に、その演出区間（連鎖開始 〜 演出終了、最大 10 秒）を自動的に
mp4 として端末に保存する。本機能は EndlessScreen 等のプレイ画面側のフックと、共通基盤_ChainClipRecorder の組み合わせで実装する。

## 画面要素

- プレイ画面の右上に「保存しました」トースト（`chain_clip_saved_toast`）
- 設定画面で機能 ON/OFF（`settings_chain_clip_toggle`）

## ユーザー操作

- 5 連鎖以上を達成 → 演出区間を自動キャプチャしギャラリーに保存
- トーストから「シェア」ボタンで OS 標準シェアシートを開ける（`chain_clip_share_action`）

## エラーケース

- ストレージ容量不足: 警告トーストを表示しキャプチャを中止（プレイ自体は継続）
- 端末性能不足: 保存を断念し警告ログ
- 権限拒否: 設定で機能を OFF にする旨を案内

## データモデル

PlayViewModel が ChainEngine の Flow<ChainEvent> を購読し `level >= 5` のとき
`ChainClipRecorder.startCaptureIfBigChain(event, surface)` を呼ぶ。完了時に Toast `chain_clip_saved_toast` を表示。

## 受け入れ条件

- Given: 設定で大連鎖クリップ自動保存 ON
- When: プレイ中に 5 連鎖を発生させる
- Then: ギャラリーに動画ファイル `美少女連鎖パズル_<timestamp>.mp4` が保存され、画面に保存トーストが表示される（Maestro: MediaStore クエリで対象ファイルの存在確認）。
