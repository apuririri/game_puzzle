# セーブ／ロード（SaveSlotScreen / オート＋手動スロット）

## 機能概要

中断復帰用のオートセーブ（共通基盤_SaveLoadManager が管轄）と、複数スロットの手動セーブを UI から操作する画面。
プレイ画面の一時停止メニュー、設定画面から起動される。

## 画面要素

- 画面ルート（`save_slot_root`）
- スロット一覧画面（`save_slot_list`）
- 各スロット行（`save_slot_{index}`、index ∈ 1..10）
- 「上書き」ボタン（`save_slot_overwrite_button`）
- 「読み込み」ボタン（`save_slot_load_button`）
- 「削除」ボタン（`save_slot_delete_button`）
- 上書き確認ダイアログ（`save_slot_overwrite_dialog`）
- 「戻る」ボタン（`save_slot_back_button`）

## ユーザー操作

- 一覧から任意スロットを選択 → 「上書き」「読み込み」「削除」のいずれか
- 上書き時は確認ダイアログ表示
- 読み込み成功 → 呼び出し元プレイ画面に snapshot 復帰

## エラーケース

- セーブデータ破損: ダイアログ `save_slot_corrupt_dialog` で通知し、当該データを破棄して新規プレイ開始へ誘導。
- スロットが空: 「読み込み」「削除」をグレーアウト。

## データモデル

呼び出し側 `source` クエリパラメータで動作モードを区別:
- `source=play_pause`（プレイ中の一時停止メニュー）→ 上書き／読み込み可
- `source=settings`（設定経由）→ 削除のみ

## 受け入れ条件

- Given: エンドレスモード中にホームへ戻る
- When: 再度アプリを開き「つづきから」相当の復帰を行う
- Then: 直前のぷよ配置・スコア・連鎖状態が復元される（共通基盤_SaveLoadManager の autoLoad + 本画面の manual load も検証）。
