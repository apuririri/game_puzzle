# リザルト / ゲームオーバー画面（ResultScreen）

## 機能概要

プレイ終了時にスコア・最大連鎖数・使用キャラ・TOP10 入り判定を表示する画面。
シェアボタン・もう一度・タイトルへ の 3 動作を提供する。

## 画面要素

- 画面ルート（`result_root`）
- 最終スコア表示（`result_score_text`）
- 最大連鎖数表示（`result_max_chain_text`）
- 使用キャラ立ち絵（`result_character_image`）
- 「シェア」ボタン（`result_share_button`）
- 「もう一度」ボタン（`result_retry_button`）
- 「タイトルへ」ボタン（`result_back_to_title_button`）
- TOP10 入り時の演出（`result_topten_banner`）

## ユーザー操作

- 「もう一度」押下 → 同一モードを再開（`mode` をクエリから読んで再 navigate）
- 「タイトルへ」押下 → タイトル画面へ戻る（`popUpTo("title") { inclusive = false }`）
- 「シェア」押下 → スクリーンショット共有（ScreenCapture）

## エラーケース

- スコア記録時に Room 書き込み失敗: スナックバー `result_save_error_snackbar` で通知し、ボタン操作は継続可能とする。

## データモデル

- `ResultUiState(mode, score, maxChain, characterId, isTop10, rank: Int?)`
- 起動時に `RankingRepository.insertIfTop10(mode, score, maxChain, characterId, playedAt)` を呼ぶ。

## 受け入れ条件

- Given: スコアアタックモードを最後までプレイ
- When: 制限時間 0 でリザルト画面に遷移
- Then: スコア・最大連鎖数・使用キャラ立ち絵が表示され、TOP10 入り判定が反映される（`result_topten_banner` の表示有無を Compose UI Test）。
