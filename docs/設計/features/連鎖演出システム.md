# 連鎖演出システム（プレイ画面統合 / 共通エンジン呼び出し）

## 機能概要

連鎖が発生した際に、画面全体で爽快感を最大化する演出を発火する。各プレイ画面（エンドレス／スコアアタック／ストーリー／CPU 対戦）から呼び出される。
本ファイルは「プレイ画面が連鎖演出をどう統合するか」を定義する（描画コア実装は 共通基盤_ChainEffectRenderer）。

## 画面要素

各プレイ画面に重ねる以下のオーバーレイ要素（実体は共通基盤_ChainEffectRenderer の testTag）:

- `chain_particle_layer`（消去ぷよ拡大 + パーティクル）
- `chain_count_overlay`（連鎖数ズーム）
- `chain_screen_shake`（画面シェイク）
- `chain_background_flash`（背景フラッシュ／カラーグラデ）
- `chain_character_overlay`（キャラ立ち絵差し替え: 通常 → 連鎖中 → 大連鎖）
- `chain_bgm_intensify`（5 連鎖以上で BGM 高揚スイッチ）

## ユーザー操作

- 連鎖判定がトリガされる → 上記演出が並列発火し、消去アニメーション完了後に次のぷよ落下を再開
- 演出は短時間でループせず、連鎖ごとに段階的に派手さが増す（level に応じてパーティクル数・シェイク強度・フラッシュ alpha がスケール）

## エラーケース

- 端末性能不足でフレームレート低下: ChainEffectConfig.particleCount を 30fps を下回らない範囲で動的に削減
- BGM スイッチ中に連鎖終了: 元の BGM パートにフェードバック

## データモデル

PlayViewModel が ChainEngine の `Flow<ChainEvent>` を ChainEffectOverlay に渡す。`level >= 5` で
AudioManager.play(AudioCue.Bgm("play_intense")) を発火、`<5` で `play_normal` を維持。

連鎖ボイス: level に応じて `chain_1` / `chain_2` / `chain_big`（5+）を `AudioCue.Voice(selectedCharacterId, …)` で再生。
連鎖 SE: level に応じて `pop_small`（1-2）／`pop_big`（3+）を発火。

## 受け入れ条件

- Given: 任意のプレイ画面で 5 連鎖以上を発生させる
- When: 連鎖判定が走る
- Then: 拡大パーティクル・画面シェイク・大連鎖ボイス・大連鎖 SE・BGM 高揚スイッチが連動して発火する（Maestro + ADB スクショ で確認、logcat に `UiLog::chainEffect level=5` が出る）。
