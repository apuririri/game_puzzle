# スコアアタックモード（ScoreAttackScreen）

## 機能概要

制限時間（3 分）内のハイスコアを競うモード。制限時間終了でリザルト画面へ自動遷移しスコアを記録する。
プレイ UI はエンドレスモードと共通（EndlessScreen の `playMode` パラメータ違い）。

## 画面要素

- 画面ルート（`score_attack_root`）
- エンドレスモードと同じプレイ画面 UI（`playfield_*`）
- 残り時間表示（`scoreattack_timer_text`）

## ユーザー操作

- 開始から制限時間カウントダウン
- 操作・連鎖ロジックはエンドレスと共通
- 残り時間 0 → リザルト画面（`result?mode=scoreAttack&...`）へ自動遷移し、TOP10 入りすればローカルランキングに記録

## エラーケース

- 一時停止中はタイマー停止
- バックグラウンド復帰時もタイマーは停止扱い
- プロセス再起動: セッションは破棄され、タイトル画面へ戻る（中断スコアは保存しない）

## データモデル

- `ScoreAttackUiState(remainingMs: Long, ...)` 内部で 1tick = 16ms

## 受け入れ条件

- Given: スコアアタックモードを開始
- When: 制限時間が経過する
- Then: 自動でリザルト画面に遷移し、スコアと連鎖数がランキング判定に渡される（Compose UI Test で 5 秒設定の short モードを使い検証）。
