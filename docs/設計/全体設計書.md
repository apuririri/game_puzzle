# 全体設計書（美少女連鎖パズル）

> 本書はシステムループ S4 で作成された初版である（FR-5.4 / 設計書は実装に同期する地図だが、初版のみ実装前に作成可）。
> 本書から派生する各機能設計書は `docs/設計/features/<機能名>.md` に配置する。
> ルール: 食い違ったらコードが正、検出は `autodev/scripts/check_design_sync.sh`。

## 1. システム全体像

- クライアント単体（Kotlin + Jetpack Compose / Material 3）。縦持ち固定。
- ゲームロジック・描画（連鎖エフェクト）・サウンド・セーブ／ロード・ローカルランキングをすべて端末内で完結。
- 外部依存: 標準シェアシート（ACTION_SEND）のみ。バックエンド・通信なし（INTERNET 権限不要）。
- アセット（画像 35 枚 / 音声多数）は `assets/` にバンドル済み。生成パイプラインは開発時のみ（ComfyUI / VoiceBOX / Docker）。
- 永続化: Room（ランキング・進行状況・セーブ）+ DataStore（設定・選択キャラ・最終チュートリアルページ）。

## 2. 機能間関係マップ

```
[Title] ── start ──▶ [ModeSelect] ──┬─▶ [Endless]
                                    ├─▶ [ScoreAttack]
                                    ├─▶ [Story] ── battle ──▶ [Endless系プレイ画面]
                                    └─▶ [CpuBattle]
[Title] ── characterSelect ──▶ [CharacterSelect]
[Title] ── ranking ──▶ [Ranking]
[Title] ── tutorial ──▶ [Tutorial]
[Title] ── settings ──▶ [Settings]
[Endless/ScoreAttack/Story/CpuBattle] ── end ──▶ [Result]
[Result] ── retry ──▶ same mode  /  back ──▶ [Title]  /  share ──▶ OS ShareSheet

共通エンジン（縦串）:
  ChainEngine            （連鎖判定 + 演出トリガ）
  AudioManager           （BGM/SE/Voice の AudioFocus 管理 + 設定反映）
  ChainEffectRenderer    （パーティクル・シェイク・フラッシュ）
  ChainClipRecorder      （5連鎖以上の自動キャプチャ）
  ScreenCapture          （シェア用スクリーンショット）
  SaveLoadManager        （オート+手動セーブ。Room）
  RankingRepository      （HighScore TOP10。Room）
  SettingsRepository     （DataStore）
  CharacterRepository    （CharacterEntity + CharacterImageEntity。プリロード）
  StoryProgressRepository（StoryProgress。Room）
```

| ルート | Composable | 引数 | 概要 |
|---|---|---|---|

| `character_select` | CharacterSelectScreen | - | 6 キャラから選択 |

| `play/score_attack` | ScoreAttackScreen | - | スコアアタック |

## 4. データフロー（高レベル）

```
User Input (Compose ボタン / Maestro / Compose UI Test)
     │
     ▼
ViewModel (StateFlow<UiState>)
     │  ──collect──▶ Compose Screen 再描画
     │
     ▼
UseCase ──▶ Repository ──▶ Room or DataStore
     │            │
     │            └──▶ AssetManager (assets/image|voice|bgm|se)
     ▼
ChainEngine (純粋ロジック / GameField, Pair, Cell)
     │
     ├──▶ ChainEffectRenderer (Canvas + animateFloatAsState)
     ├──▶ AudioManager (MediaPlayer / SoundPool)
     ├──▶ ChainClipRecorder (MediaCodec + MediaMuxer)
     └──▶ SaveLoadManager (oneshot save on every land)
```

## 5. Room データモデル（要件 §5 を全反映）

DB 名: `app_database`（Room）。schema 出力: `app/schemas/com.example.myapp.data.local.AppDatabase/`。

### 5.1 Entity 一覧

| Entity | 主キー | 用途 | 主なカラム |
|---|---|---|---|
| `CharacterEntity` | id (String) | キャラマスタ | displayName, voiceTone, unlocked |
| `CharacterImageEntity` | id (String) | キャラ立ち絵バリエーション | characterId(FK), variant, assetPath |
| `HighScoreEntity` | id (Long, autoGenerate) | ランキング TOP10 | mode, score, maxChain, characterId, playedAt |
| `StoryProgressEntity` | id = characterId | ストーリー進行 | clearedChapter, updatedAt |
| `SaveSlotEntity` | slotIndex (Int) | 手動スロット | mode, serializedGameState, score, savedAt |
| `AutoSaveEntity` | id = 0 (Int 固定) | オートセーブ単一行 | mode?, serializedGameState?, savedAt? |

### 5.2 Variant 列挙（CharacterImageEntity.variant）

`normal` / `joy` / `anger` / `sad` / `chain` / `bigChain` / `lose`

### 5.3 mode 列挙（HighScoreEntity.mode / SaveSlotEntity.mode / AutoSaveEntity.mode）

`endless` / `scoreAttack` / `cpuBattle`
※ストーリーモードはランキング対象外（StoryProgressEntity のみ）。

### 5.4 Migration 方針

- v1 で全 6 Entity を一括導入。
- 以降はカラム追加時のみ +1 し、Migration クラス + MigrationTest（androidTest）を必須。
- `check_room_schema.sh` で schema 出力の追従を機械検査。

- 画像: 単色ベタ塗り + キャラ名・variant 名テキスト合成（`assets/image/character/.../normal.webp` 等）

- 実アセット差替えは納品時タスクとして `CHANGELOG.md` に明記する。

## 8. 共通機能（縦串）

### 8.1 ChainEngine（純粋ロジック / 共通）

- `GameField(6 cols, 14 rows)` を 2D 配列で保持
- `Pair`（落下中ペア）、`Cell`（色 = 5 色 + ojama）
- `step(input)`: 入力反映 → 着地判定 → 連鎖判定（同色 3 つ以上の縦横連結を消去 → 重力 → 再帰）
- 連鎖発生時は `ChainEvent(level=n, color=...)` を Flow に流す
- 全プレイ画面が共有（Endless / ScoreAttack / Story battle / CpuBattle）

### 8.2 AudioManager

- BGM: MediaPlayer 1〜2 個（クロスフェード）
- SE: SoundPool（同時多発前提）
- Voice: MediaPlayer（直前ボイス即停止 → 新規）
- AudioFocus 喪失 → 通話復帰時に BGM のみ再開（SE/Voice はスキップ）
- 設定 ON/OFF・音量は `SettingsRepository.observe()` で即時反映

### 8.3 ChainEffectRenderer

- パーティクル: Compose Canvas + `animateFloatAsState`（端末性能不足時はパーティクル数を 30fps 維持する範囲で削減）
- 画面シェイク: `Modifier.offset { translation }`
- カラーフラッシュ: 全画面 alpha 矩形オーバーレイ
- 連鎖数ズーム: `chain_count_overlay` Composable

### 8.4 ChainClipRecorder

- MediaCodec H.264 + MediaMuxer で mp4（10 秒上限）
- 5 連鎖以上検出時に「直前 1 秒〜演出終了」を切り出して `MediaStore.Video` に保存
- 失敗時はトーストで通知してスキップ（fallback 禁止: 別形式に勝手に切り替えない）

### 8.5 ScreenCapture

- `View.drawToBitmap()` でビットマップ取得 → FileProvider 経由で `ACTION_SEND`
- 失敗時は再試行案内

### 8.6 SaveLoadManager

- オートセーブ: 着地ごとに `AutoSaveEntity(id=0)` を上書き
- 手動セーブ: `SaveSlotEntity(slotIndex=1〜10)`
- `serializedGameState` は kotlinx.serialization JSON
- スキーマ変更時は Migration 必須

### 8.7 RankingRepository

- mode × score 上位 10 を `HighScoreEntity` から SELECT
- 同点は `playedAt ASC` で上位
- リザルト画面で TOP10 判定（is_top10 フラグを返す）

### 8.8 SettingsRepository（DataStore）

- 全キーを `Flow<AppSettings>` として公開
- 書き込み失敗時はスナックバーで通知し前値にロールバック

### 8.9 CharacterRepository

- 起動時に `CharacterEntity` × 5 と `CharacterImageEntity` × 35 を初回シード
- `AssetManager.open()` で立ち絵をデコード

### 8.10 StoryProgressRepository

- characterId 毎に `clearedChapter` を保持
- 章クリア時に更新

## 9. 共通スタイル（Material 3）

- カラースキーム: `dark` 既定（鮮やかな演出と相性）
- 主要色: `primary = HotPink (#FF6FA8)`, `secondary = Purple (#7C4DFF)`, `tertiary = Cyan (#00E5FF)`
- 文字: システム文字サイズ倍率に追従（`sp` 単位）
- ボタン: 最小タッチターゲット 48dp（P4 ペルソナの押し間違い対策）
- 演出オーバーレイ: 半透明 → 不透明トランジション、`AnimatedVisibility` を多用
- 動的状態用 Composable: `LoadingOverlay` / `EmptyState` / `ErrorState`（S11 UI 改善で発火スクショ撮影に使う）

## 10. testTag 命名方針

- 基本形式: `<画面>_<要素>_<種別>`
- 動的: `<画面>_<要素>_{key}` (例: `character_card_{characterId}`, `ranking_row_{rank}`, `save_slot_{index}`)
- 画面ルート: `<画面>_root`（NavHost composable のトップ Box に付与）
- 一覧（LazyColumn / LazyRow）: `<画面>_<要素>_list`
- ボタン: `<画面>_<要素>_button`
- 入力欄: `<画面>_<要素>_input`
- トグル: `<画面>_<要素>_toggle`
- スライダ: `<画面>_<要素>_slider`
- Maestro / Compose UI Test は **testTag のみ** で操作（座標タップ禁止 / CLAUDE_MAIN.md 規約10）

## 11. 非機能要件への対応マップ

| 要件 | 設計上の対応 |
|---|---|
| 起動 < 3秒 | TitleScreen は同期描画、CharacterRepository は LaunchedEffect で非同期シード |
| 60fps 目標 / 最低 30fps | ChainEffectRenderer がパーティクル数を実 fps を見て削減 |
| 着地 → 連鎖開始 < 100ms | ChainEngine.step は I/O なし、サスペンド関数で 1tick = 16ms 以内 |
| オフライン | 通信なし。assets/ 完結 |
| 権限最小化 | INTERNET 不要。クリップ保存のみ MediaStore（READ_MEDIA_IMAGES 不要、Insert API のみ） |
| 平文ファイル禁止 | DataStore / Room のみ |
| 文字サイズ追従 | dp ではなく sp |
| testTag 必須 | 全主要 UI に testTag |
